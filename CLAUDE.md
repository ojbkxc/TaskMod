# UI 设计参考规范

## 整体风格定位
本项目 UI 设计参考 **DeepSeek** 及社区增强项目 **DeepSeek++ (deepseek-pp)** 的设计语言[reference:0]。整体定位为 **"AI Agent 工作台"** 风格，兼具专业工具的效率与 AI 产品的沉浸感。

## 核心设计原则

### 1. 视觉基调
- **克制与耐看**：整体观感应**克制、耐看**，配色柔和协调，避免花哨[reference:2]。
- **极简主义**：采用 **minimal 主题**，弱化边框、模块分割和多余提示，把用户注意力集中在核心功能上[reference:3][reference:4]。
- **层级分明**：用 **elevated 主卡片**叠加 **outlined 内嵌卡片**拉开视觉层级[reference:5]。

### 2. 色彩系统
- **色彩收敛**：整体色彩收敛但语义清晰。使用 **primary / success / warning** 等标准语义色[reference:6]。
- **大胆点缀**：可在 Header 等关键位置使用**大胆的渐变色**（如紫色渐变）增强品牌识别度。
- **深色/浅色双模式**：需完整适配**深色与浅色两种模式**[reference:8]。

### 3. 排版与图标
- **无 Emoji 堆砌**：不使用 Emoji 作为主要图标，每个图标都应对应明确的功能含义[reference:9]。
- **信息密度**：在保证清晰的前提下，可适当提高信息密度，适合专业用户。

### 4. 交互与动效
- **卡片悬浮效果**：卡片可带有轻微的悬浮效果，增强视觉层次感。
- **沉浸式对话体验**：对话界面应保证沉浸式体验，同时能承载复杂任务与功能[reference:12]。

## 典型页面参考

### 对话界面
- 采用**居中布局**，视觉干扰降到最低[reference:13]。
- 画布中央放置 Logo 与欢迎语，**留白比例较大**[reference:14]。
- 输入框下方直接呈现**高频能力入口**（如工具、技能等），不堆叠说明文案[reference:15]。

### 侧边栏
- 侧边栏用于**历史对话记录**、工具管理和设置入口[reference:16]。
- 历史记录以**列表窗口**形式嵌入，支持搜索、编辑、删除[reference:18]。

### 设置面板
- 采用**弹窗形式**呈现，避免跳转打断当前操作[reference:19]。
- 左侧为**垂直导航栏**，信息架构清晰[reference:20]。

## 技术实现要求
- 使用 **Dioxus** 框架和 **eq_ui** 组件库实现上述设计。
- 组件样式需支持**深色/浅色主题**的动态切换。
- 遵循 **eq_ui** 的原子设计规范，保持组件的一致性和可维护性。
